# 1️⃣ 연결 자료구조

## 📌 개념
- 데이터를 **연속되지 않은 메모리에 저장하고 포인터로 연결하는 구조**

👉 순차 자료구조와 반대 개념 :contentReference[oaicite:0]{index=0}  

---

## 📌 특징

✔ 메모리 비연속 저장  
✔ 동적 크기 조절 가능  
✔ 삽입/삭제 효율적  

❌ 접근 속도 느림 (순차 탐색 필요)

---

# 2️⃣ 연결 리스트 (Linked List)

## 📌 개념
- 각 노드(Node)가 **데이터 + 포인터**로 구성됨

---

## 📌 기본 구조
~~~c
typedef struct Node {
    int data;
    struct Node* link;
} Node;
~~~

---

## 📌 구성 요소

✔ 데이터 필드  
✔ 링크 필드 (다음 노드 주소)

---

## 📌 형태
~~~text
[데이터 | 주소] → [데이터 | 주소] → NULL
~~~

---

# 3️⃣ 연결 리스트 종류

---

## 📌 단순 연결 리스트 (Singly Linked List)
- 한 방향 연결
- 가장 기본 구조

---

## 📌 원형 연결 리스트 (Circular Linked List)
- 마지막 노드 → 첫 노드 연결
- NULL 없음

---

## 📌 이중 연결 리스트 (Doubly Linked List)
~~~c
struct Node {
    int data;
    struct Node* llink;
    struct Node* rlink;
};
~~~

- 양방향 이동 가능

---

# 4️⃣ 연결 리스트 연산

---

## 📌 노드 생성
~~~c
Node* newNode = (Node*)malloc(sizeof(Node));
~~~

---

## 📌 삽입 (Insert)

### ✔ 처음에 삽입
~~~c
newNode->link = head;
head = newNode;
~~~

---

### ✔ 중간 삽입
1. 이전 노드 찾기
2. 링크 재설정

~~~c
newNode->link = prev->link;
prev->link = newNode;
~~~

---

### ✔ 끝에 삽입
- 마지막 노드까지 이동 후 연결

---

## 📌 삭제 (Delete)

### ✔ 첫 노드 삭제
~~~c
Node* temp = head;
head = head->link;
free(temp);
~~~

---

### ✔ 중간 삭제
~~~c
prev->link = delNode->link;
free(delNode);
~~~

---

# 5️⃣ 연결 리스트 순회 (Traversal)

~~~c
Node* p = head;
while(p != NULL){
    printf("%d", p->data);
    p = p->link;
}
~~~

# 6️⃣ 연결 리스트 vs 배열

| 구분 | 배열 | 연결 리스트 |
|------|------|------------|
| 메모리 | 연속 | 비연속 |
| 접근 | 빠름 | 느림 |
| 삽입/삭제 | 느림 | 빠름 |
| 크기 | 고정 | 가변 |

---

# 7️⃣ 연결 리스트 핵심 포인트

✔ 포인터 기반 구조  
✔ 삽입/삭제 시 주소 변경  
✔ 메모리 동적 할당 필수  

---
