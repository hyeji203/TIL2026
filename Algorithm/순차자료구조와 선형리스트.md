# 1️⃣ 순차 자료구조

## 📌 개념
- 데이터를 **논리적 순서대로 메모리에 연속 저장하는 구조**
- 논리적 순서 = 물리적 순서 (중요)

👉 배열(Array)로 구현됨 :contentReference[oaicite:0]{index=0}  

---

## 📌 특징
✔ 메모리 연속 저장  
✔ 접근 속도 빠름 (인덱스 기반)  
❌ 삽입/삭제 비용 큼  

---

# 2️⃣ 선형 리스트 (Linear List)

## 📌 개념
- 자료들이 **순서를 가지며 나열된 구조**

---

## 📌 종류

### ✔ 순차 리스트
- 배열 기반
- 연속된 메모리

### ✔ 연결 리스트
- 포인터 기반
- 비연속 메모리

👉 일반적으로 "선형 리스트" = 순차 리스트 의미

---

## 📌 배열 표현
~~~c
int sale[4] = {157, 209, 251, 312};
~~~

- 메모리에 연속 저장
- 주소 계산 가능

---

# 3️⃣ 배열의 메모리 구조

## 📌 1차원 배열 주소 계산
👉 주소 = 시작주소 + (인덱스 × 크기)

---

## 📌 2차원 배열

### ✔ 행 우선 방식 (C 언어 기본)
👉 주소 = α + (i × 열개수 + j) × 크기

---

### ✔ 열 우선 방식
👉 주소 = α + (j × 행개수 + i) × 크기

---

## 📌 3차원 배열

### ✔ 면 우선 방식
👉 주소 = α + {(i × nj × nk) + (j × nk) + k} × 크기

---

# 4️⃣ 선형 리스트 연산

---

## 📌 원소 삽입

### ✔ 과정
1. 삽입 위치 이후 데이터 뒤로 이동
2. 빈 자리에 값 삽입

---

### ✔ 이동 횟수
👉 n - k + 1

(마지막 인덱스 - 삽입 위치 + 1)

---

### ✔ 핵심 포인트
- 뒤에서부터 이동해야 데이터 덮어쓰기 방지

---

## 📌 원소 삭제

### ✔ 과정
1. 원소 삭제
2. 뒤 원소 앞으로 이동

---

### ✔ 이동 횟수
👉 n - k

(마지막 인덱스 - 삭제 위치)

---

### ✔ 핵심 포인트
- 앞에서부터 이동

---

# 5️⃣ 선형 리스트 프로그램 핵심

✔ 삽입 시 → 뒤로 이동  
✔ 삭제 시 → 앞으로 이동  
✔ 배열 크기 초과 주의  

---

# 6️⃣ 행렬의 선형 리스트 표현

## 📌 행렬 개념
- 행(row)과 열(column)로 구성

---

## 📌 종류
✔ 정방행렬 → 행 = 열  
✔ 전치행렬 → 행과 열 교환  

---

# 7️⃣ 희소 행렬 (Sparse Matrix)

## 📌 문제점
- 0이 많아 **메모리 낭비 발생**

---

## 📌 해결 방법
👉 0이 아닌 값만 저장

---

## 📌 저장 방식
~~~text
(행, 열, 값)
~~~

예:
~~~text
<0,2,2>
<1,4,7>
<2,0,23>
~~~

---

## 📌 특징
✔ 메모리 효율 증가  
✔ 처리 속도 개선  

---

# 8️⃣ 희소 행렬 연산

## 📌 전치 행렬
- 행 ↔ 열 교환

---

# 9️⃣ 다항식 표현

## 📌 개념
👉 여러 항의 합

~~~text
A(x) = 4x^3 + 3x^2 + 2
~~~

---

## 📌 특징
✔ 지수 기준 내림차순 정렬  
✔ 차수 = 가장 큰 지수  

---

## 📌 문제점
- 배열 사용 시 공간 낭비 발생

---

## 📌 해결
👉 필요한 항만 저장

---

# 🔟 다항식 연산

## 📌 덧셈
- 같은 지수끼리 더함

---

## 📌 핵심 로직
1. 지수 비교
2. 큰 지수부터 처리
3. 같은 경우 합 계산

---
