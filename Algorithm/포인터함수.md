## 1. 포인터란?

> **포인터(pointer)**: 주소를 가지고 있는 변수

- 변수는 메모리에 저장된다.
- 메모리는 **바이트 단위**로 액세스된다.
- 첫 번째 바이트의 주소는 0, 두 번째 바이트는 1, …

### 변수와 메모리 크기

| 자료형 | 크기 |
|--------|------|
| `char` | 1 바이트 |
| `short` | 2 바이트 |
| `int` | 4 바이트 |
| `float` | 4 바이트 |
| `double` | 8 바이트 |

---

## 2. 변수의 주소

- 변수의 주소를 계산하는 연산자: `&`
- 변수 `i`의 주소: `&i`

```c
int main(void) {
    int i = 10;
    char c = 69;
    float f = 12.3;
    printf("i의 주소: %p\n", &i);
    printf("c의 주소: %p\n", &c);
    printf("f의 주소: %p\n", &f);
    return 0;
}
```

> ⚠️ 프로그램을 실행할 때마다 주소는 달라진다.

---

## 3. 포인터의 선언

```c
int *p;  // 정수를 가리키는 포인터 p 선언
```

- 포인터 변수 선언 시 가리키는 타입을 명시해야 한다.

### 다양한 포인터 선언 예시

```c
char c = 'A';
float f = 36.5;
double d = 3.141592;

char   *pc = &c;   // 문자를 가리키는 포인터
float  *pf = &f;   // 실수를 가리키는 포인터
double *pd = &d;   // 실수를 가리키는 포인터
```

### ⚠️ 주의: 여러 포인터를 한 줄에 선언할 때

```c
int *p1, p2, p3;    // ❌ 잘못됨: p2, p3는 정수형 변수가 됨
int *p1, *p2, *p3;  // ✅ 올바름: 모두 포인터 변수
```

---

## 4. 포인터 초기화

### 변수의 주소로 초기화

```c
int i = 10;
int *p;
p = &i;  // 변수 i의 주소를 포인터 p에 대입
```

### NULL 포인터

```c
int *p = NULL;  // 아무것도 가리키지 않음을 명시
```

- `NULL`은 `((void *)0)`으로 정의된 포인터 상수
- 초기값이 없을 때는 반드시 `NULL`로 초기화할 것

> ⚠️ 절대 주소 초기화(`char *p = (char *) 0x30000000`)는 아두이노 같은 임베디드 환경에서만 사용 가능

---

## 5. 간접 참조 연산자 (`*`)

> `*p` : 포인터 `p`가 가리키는 위치의 값을 가져온다.

```c
int i = 10;
int *p = &i;
printf("%d\n", *p);  // 출력: 10

*p = 20;             // 포인터를 통해 변수 값 변경
printf("%d\n", i);   // 출력: 20
```

### `&` 연산자 vs `*` 연산자

| 연산자 | 의미 | 사용 시점 |
|--------|------|-----------|
| `&i` | 변수 i의 주소 반환 | 포인터에 주소 대입할 때 |
| `*p` | 포인터 p가 가리키는 값 반환 | 포인터로 간접 참조할 때 |

---

## 6. 포인터 사용 시 주의점

1. **초기화되지 않은 포인터 사용 금지**
   ```c
   int *p;
   *p = 100;  // ❌ 위험! 쓰레기 주소에 접근
   ```

2. **포인터의 타입과 변수의 타입은 반드시 일치**
   ```c
   int i;
   double *pd = &i;  // ❌ 타입 불일치
   ```

3. **아무것도 가리키지 않을 때는 NULL로 초기화**

> "With great power comes great responsibility" — 포인터는 C언어의 강점이자 단점이다.

## 7. 포인터 연산

가능한 연산: **증가(`++`), 감소(`--`), 덧셈(`+`), 뺄셈(`-`)**

포인터에 1을 더하면 **가리키는 타입의 크기**만큼 주소가 증가한다.

| 포인터 타입 | `++` 연산 후 증가값 |
|-------------|---------------------|
| `char *` | 1 |
| `short *` | 2 |
| `int *` | 4 |
| `float *` | 4 |
| `double *` | 8 |

```c
char   *pc = (char   *)10000;
int    *pi = (int    *)10000;
double *pd = (double *)10000;

// pc+1 = 10001, pi+1 = 10004, pd+1 = 10008
```

### 간접 참조 연산자와 증감 연산자 조합

| 수식 | 의미 |
|------|------|
| `v = *p++` | p가 가리키는 값을 v에 대입 후, p를 증가 |
| `v = (*p)++` | p가 가리키는 값을 v에 대입 후, 그 값을 증가 |
| `v = *++p` | p를 증가 후, p가 가리키는 값을 v에 대입 |
| `v = ++*p` | p가 가리키는 값을 증가하여 v에 대입 |

## 8. 포인터와 함수

### 인수 전달 방법

| 방식 | 설명 | 특징 |
|------|------|------|
| **값에 의한 호출** (call by value) | 복사본 전달 | 원본 변경 불가 |
| **참조에 의한 호출** (call by reference) | 주소 전달 | 원본 변경 가능 |

### 값에 의한 호출 (실패 예)

```c
void swap(int x, int y) {
    int tmp = x; x = y; y = tmp;  // 원본 a, b는 변경되지 않음
}
```

### 참조에 의한 호출 (포인터 활용)

```c
void swap(int *px, int *py) {
    int tmp = *px;
    *px = *py;
    *py = tmp;
}

// 호출: swap(&a, &b);
```

### 포인터로 여러 값 반환

```c
int get_line_parameter(int x1, int y1, int x2, int y2,
                        float *slope, float *yintercept) {
    if (x1 == x2) return -1;
    *slope = (float)(y2 - y1) / (float)(x2 - x1);
    *yintercept = y1 - (*slope) * x1;
    return 0;
}
```

### ⚠️ 지역 변수의 주소 반환 금지

```c
int *add(int x, int y) {
    int result = x + y;
    return &result;  // ❌ 함수 종료 후 사라지는 변수의 주소!
}
```

### `const` 포인터 매개변수

함수 내에서 값 변경을 막으려면:

```c
void sub(const int *p) {
    *p = 0;  // ❌ 오류: const 포인터는 값 변경 불가
}
```

## 9. 포인터와 배열

- **배열 이름은 포인터**다 (첫 번째 원소의 주소)
- 포인터는 배열처럼 사용 가능

```c
int a[] = {10, 20, 30, 40, 50};

// 아래 표현들은 모두 동일
a[0] == *a
a[1] == *(a+1)
&a[0] == a
```

### 포인터를 배열처럼 사용

```c
int *p = a;
p[0] = 60;  // a[0]도 60으로 변경됨
p[1] = 70;
```

### 배열을 함수에 전달

배열은 복사 없이 **주소(포인터)만 전달**된다.

```c
// 아래 두 선언은 동일
void sub(int b[], int size)
void sub(int *b, int size)
```

```c
void sub(int b[], int n) {
    b[0] = 4;  // 원본 배열 변경 가능
    b[1] = 5;
    b[2] = 6;
}
```

## 10. 포인터 형변환

꼭 필요한 경우 명시적으로 포인터 타입을 변환할 수 있다.

```c
double *pd = &f;
int *pi;
pi = (int *)pd;  // 명시적 형변환
```

## 11. 포인터 사용의 장점

1. **연결 리스트, 이진 트리** 등 고급 자료구조 구현 가능
2. **참조에 의한 호출**로 함수 외부 변수 값 변경 가능
3. **메모리 매핑 하드웨어** 접근 가능 (임베디드 시스템)
4. **동적 메모리 할당** 사용 가능 (17장)

## 📝 중간 점검 정리

| 질문 | 답 |
|------|----|
| 메모리 주소 단위 | 바이트(byte) |
| 변수 주소 추출 연산자 | `&` |
| 간접 참조 연산자 | `*` |
| 초기화 안 된 포인터에 넣을 값 | `NULL` |
| 포인터에 가능한 연산 | `++`, `--`, `+`, `-` |
| int 포인터가 80번지일 때 `p+1` | 84번지 |
| 배열을 함수에 전달 시 복사 여부 | 복사 없음 (주소만 전달) |
