## 클로드 코드 실전 활용 (할 일 관리 앱 만들기)

---

## 🎯 학습 목표
- PRD를 단계별 프롬프트로 변환하여 앱 구현
- 세션 및 컨텍스트 관리 이해
- 프로젝트 개선 및 관리 방법 학습

---

# 1️⃣ 핵심 개념 정리

## 📌 단계별 프롬프트 전략
- PRD를 여러 단계로 나누어 개발 진행
- 한 번에 만들지 않고 **단계적으로 구현**

👉 이유
- 품질 향상
- 오류 감소
- 점진적으로 기능 추가

---

## 📌 PRD → 단계별 프롬프트 변환 요청
- 제품 요구사항 문서
- 클로드가 제안하는 5단계
  Step 1: 기본 구조 및 핵심 기능 구현
  Step 2: 카테고리 기능 및 UI 개선
  Step 3: 진행률 대시보드 추가
  Step 4: 다크 모드 및 고급 기능
  Step 5: 최종 완성 및 최적화
---
## 📌 컴퓨팅 사고 적용
- 분해 → 문제 나누기
- 패턴 인식 → 반복 구조 찾기
- 추상화 → 핵심만 정리
- 알고리즘 → 순서대로 해결
---

# 2️⃣ 할 일 관리 앱 만들기 흐름

## 📌 전체 개발 단계
### Step 1: 기본 기능
- HTML / CSS / JS 구조 생성
- 할 일 추가 / 삭제 / 체크
- localStorage 저장
---
### Step 2: 카테고리 기능
- 업무 / 개인 / 공부 분류
- 색상 태그
- 필터 기능
---
### Step 3: 진행률 대시보드
- 전체 진행률 표시 (예: 5/10, 50%)
- 프로그레스 바
- 인라인 수정 기능
---
### Step 4: 고급 기능
- 다크 모드
- 완료 항목 삭제
- 단축키 지원
---
### Step 5: 최적화
- JSON 저장 / 불러오기
- 정렬 기능
- 성능 개선
---

# 3️⃣ 클로드 코드 사용 흐름
## 📌 프로젝트 시작
~~~bash
mkdir Study-02
cd Study-02
claude
/init
~~~
---
## 📌 단계별 실행
- Step별 구현 → 확인 → 다음 단계 반복
👉 반복 구조 중요

---

# 4️⃣ 세션 관리

## 📌 세션이란?
- 클로드 코드에서 대화를 시작해 명령을 주고받는 동안 유지되는 상태

---

## 📌 세션 재개
- 이전 대화 기록을 불러와서 마지막 상태 그대로 이어서 작업
~~~bash
claude --resume
~~~

- 가장 최근 대화 복구
~~~bash
claude --continue
~~~

---

## 📌 중요 포인트
- 폴더 위치 중요 (같은 폴더에서 실행)
- 장기 프로젝트 필수 기능

---

# 5️⃣ 컨텍스트 관리

## 📌 컨텍스트란?
- 현재 대화 + 작업 정보 저장 및 표시한 메모리 공간

---

## 📌 상태 확인
~~~bash
/context
~~~

---

## 📌 컨텍스트 압축
~~~bash
/compact
~~~

👉 대화 길어질 때 필수

---

# 6️⃣ 생산성 기능

## 📌 자동 승인 모드
- Shift + Tab
- 승인 클릭 생략 → 속도 향상

---

## 📌 이미지 기반 UI 개선
- 캡처 후 드래그 & 드롭
- AI가 UI 개선 제안

---

## 📌 @ 기호 활용
~~~text
@script.js 분석해줘
@web_version 폴더 확인해줘
~~~

👉 코드 빠르게 분석 가능

---

# 7️⃣ 주요 명령어 정리

## 📌 설정 확인 및 변경
~~~bash
/config
~~~

---

## 📌 시스템 진단
~~~bash
/doctor
~~~

---

## 📌 대화 저장
~~~bash
/export
~~~

---

## 📌 추가 명령어
- /clear → 대화 초기화
- /cost → 토큰 확인
- /diff → 변경사항 확인
- /memory → 메모리 관리
- /rename → 세션 이름 변경
