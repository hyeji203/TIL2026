# [2주차] AI를 깨우는 프롬프트의 비밀과 마케팅 포트폴리오 완성

## 🎯 1. 수업 핵심 역량
* **바이브 코딩**: AI 코딩 파트너를 활용하여 자연어 프롬프트로 빠르게 개발하는 방식이다. (이번 주 적용: 클로드에게 프롬프트로 포트폴리오 웹페이지 생성 )
* **컴퓨팅 사고**: 문제를 분해하고, 패턴을 인식하며, 추상화하여 알고리즘적으로 해결하는 능력이다. (이번 주 적용: 5W1H 분해, PRD 추상화, 4단계 알고리즘 )
* **프로젝트 관리**: 요구사항을 정의하고, 일정 계획과 중간 보고를 거쳐 최종 발표에 이르는 과정이다. (이번 주 적용: PRD로 요구사항 정의, 4단계 일정 계획 )

## 💡 2. 좋은 프롬프트 작성법과 PRD
AI의 잠재력을 최대한 끌어내려면 명확하고 구체적인 프롬프트가 필수적이다. 프롬프트는 내비게이션에 목적지를 입력하는 것과 같다.

### 좋은 프롬프트의 3가지 조건
| 조건 | 설명 | 예시 |
| :--- | :--- | :--- |
| **명확성** | 모호한 표현을 피하고, 원하는 결과를 구체적으로 전달한다. | "멋진" → "밝고 깔끔한 스타일"  |
| **구체성** | 결과물에 필요한 기능이나 조건을 세부적으로 지시한다. | "시계와 검색바도 추가해 줘"  |
| **맥락 제공** | 왜 필요한지, 어떤 상황에서 활용되는지 알려준다. | "취업용 마케팅 포트폴리오야"  |

### 5W1H와 PRD(제품 기획서)
* 개발자들이 코딩 전 작성하는 **제품 기획서(PRD: Product Requirements Document)**와 좋은 프롬프트는 본질적으로 같다.
* **5W1H(Why, Who, What, When, Where, How)** 체계적 사고의 틀을 활용하여 프로젝트를 6가지 질문으로 분해할 수 있다.
* 소프트웨어 기능과 요구사항 명세 시, 특히 **What(무엇을 만들 것인가)**과 **How(어떻게 만들 것인가)**가 강조된다.

## 🛠️ 3. 4단계 포트폴리오 제작 전략
큰 프로젝트를 한 번에 요청하지 않고, 컴퓨팅 사고의 '분해'와 '알고리즘'을 적용하여 4단계로 나누어 AI에게 요청한다. 각 단계마다 '요청→생성→확인→수정'의 사이클을 반복한다.

1. **1단계 (뼈대)**: HTML 기본 구조를 잡고 섹션을 구성하며, 수정을 대비해 각 섹션에 고유 이름(ID)을 부여한다. 이 단계에서는 기능 구현 없이 구조만 요청하는 것이 핵심이다.
2. **2단계 (기능)**: 경력, 성과, 케이스 스터디 등 실제 데이터를 채워 넣다. 구체적이고 사실적인 데이터를 제공해야 AI가 의미 있는 콘텐츠를 생성한다.
3. **3단계 (디자인)**: 워드프레스 테마 같은 구체적인 레퍼런스를 제공하고, 색상과 레이아웃 등 시각적 완성도를 높인다.
4. **4단계 (점검)**: 완성된 결과물을 PRD와 대조하며 종합 품질을 검증하고, 링크 연결 및 작동 여부를 확인한다.
