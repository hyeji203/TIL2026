## 2. 클라우드 컴퓨팅이란?

### 정의

| 용어 | 정의 |
|---|---|
| **온프레미스 (On-Premise)** | 자체적인 공간과 자원을 이용해 사용자가 직접 구축·운영하는 전통적인 방식 |
| **클라우드** | 인터넷 구간 어딘가에 눈에 보이지 않는 형태로 구성된 IT 자원 집합 |
| **클라우드 컴퓨팅** | 인터넷을 통해 IT 자원을 요구에 따라 사용한 만큼 비용을 지불하는 서비스 |
| **온디맨드 (On-Demand)** | IT 자원을 사용자가 필요할 때 원하는 만큼 즉각적으로 제공하는 것 |

### 클라우드 컴퓨팅 공식

```
필요한 만큼 원할 때마다  +  인터넷으로 IT자원을  +  사용할만큼 지불하는  +  대용량 서비스  =  클라우드
```

### 클라우드 컴퓨팅의 이점

- **Agility (민첩성)**: 빠르고 혁신적으로 구축 가능
- **Elastic (탄력성)**: 손쉽게 확장 및 축소 가능
- **Cost Saving (비용 절감)**: 사용한 만큼만 지불 (Case by Case)
- 손쉬운 **글로벌 서비스** 제공
- 예상치 못한 **트래픽 폭주** 대응
- **빅데이터, 인공지능** 서비스 확장 용이

## 3. 클라우드 컴퓨팅 서비스 유형

클라우드 서비스는 제공 범위에 따라 세 가지로 구분된다.

### 3-1. IaaS (Infrastructure as a Service)

- **제공 내용**: 서버, 스토리지, 네트워크, 가상화
- **사용자 관리 범위**: OS, 미들웨어, 런타임, 데이터, 애플리케이션
- **클라우드 공급자 관리 범위**: 서버/스토리지/네트워크, 가상화

| 서버/스토리지/네트워킹 | 가상화 | 운영 체제 | 미들웨어 | 런타임 | 데이터 | 애플리케이션 |
|:---:|:---:|:---:|:---:|:---:|:---:|:---:|
| 공급자 | 공급자 | **사용자** | **사용자** | **사용자** | **사용자** | **사용자** |

**AWS 대표 IaaS 서비스**

| 서비스 | 설명 |
|---|---|
| EC2 | 가상 서버 |
| S3 | 클라우드 스토리지 |
| VPC | 가상 네트워크 |
| EBS | 블록 스토리지 |

> **활용 예시**: 스타트업이 자체 서버 없이 EC2와 S3로 웹사이트 호스팅

---

### 3-2. PaaS (Platform as a Service)

- **제공 내용**: 개발 및 배포를 위한 플랫폼
- **사용자 관리 범위**: 데이터, 애플리케이션
- **클라우드 공급자 관리 범위**: 서버/스토리지/네트워킹 ~ 런타임
- 서버 및 OS 관리 불필요

| 서버/스토리지/네트워킹 | 가상화 | 운영 체제 | 미들웨어 | 런타임 | 데이터 | 애플리케이션 |
|:---:|:---:|:---:|:---:|:---:|:---:|:---:|
| 공급자 | 공급자 | 공급자 | 공급자 | 공급자 | **사용자** | **사용자** |

**AWS 대표 PaaS 서비스**

| 서비스 | 설명 |
|---|---|
| AWS Elastic Beanstalk | 자동 배포 및 관리 |
| AWS Lambda | 서버리스 컴퓨팅 |
| Amazon RDS | 관리형 데이터베이스 서비스 |
| AWS Fargate | 컨테이너 관리 |

> **활용 예시**: 개발자가 AWS Lambda를 사용해 서버 관리 없이 코드 실행

---

### 3-3. SaaS (Software as a Service)

- **제공 내용**: 완성된 소프트웨어를 인터넷으로 제공
- **사용자 관리 범위**: 없음 (모든 것을 공급자가 관리)
- 설치 없이 이용, 유지보수 및 업데이트도 제공자가 수행

| 서버/스토리지/네트워킹 | 가상화 | 운영 체제 | 미들웨어 | 런타임 | 데이터 | 애플리케이션 |
|:---:|:---:|:---:|:---:|:---:|:---:|:---:|
| 공급자 | 공급자 | 공급자 | 공급자 | 공급자 | 공급자 | 공급자 |

**AWS 대표 SaaS 서비스**

| 서비스 | 설명 |
|---|---|
| Amazon WorkDocs | 문서 관리 및 협업 |
| Amazon Chime | 화상 회의 서비스 |
| AWS Managed Services | 완전 관리형 서비스 |

> **활용 예시**: 기업이 Amazon Chime을 통해 원격 회의 진행

**타 클라우드 SaaS 예시**: Google Workspace, GitHub, Dropbox, Microsoft 365

## 서비스 유형 한눈에 비교

| 구분 | IaaS | PaaS | SaaS |
|---|---|---|---|
| **사용자 제어 범위** | OS ~ 앱 | 데이터 ~ 앱 | 없음 |
| **공급자 제공 범위** | 하드웨어 + 가상화 | 하드웨어 ~ 런타임 | 전체 |
| **대상** | 인프라 담당자, DevOps | 개발자 | 일반 사용자 |
| **AWS 대표 예** | EC2, S3, VPC | Lambda, RDS | Chime, WorkDocs |
