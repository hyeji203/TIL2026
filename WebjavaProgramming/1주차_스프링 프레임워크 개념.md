## 🌱 본격적인 Spring Framework 소개
* **왜 이름이 '스프링(Spring)'일까?**
  과거 Java의 EJB라는 기술이 너무 복잡하고 무거워서 개발자들이 힘든 '겨울'을 보냈는데, 그 복잡성을 극복하고 개발의 **'봄'**을 가져왔다는 의미라고 한다!
* 우리나라 공공기관 웹사이트 개발의 표준이 되는 **'전자정부 표준 프레임워크'**의 핵심 기반 기술이 바로 이 스프링이라고 한다.

## ⭐️ 스프링의 4대 핵심 특징
1. **POJO (Plain Old Java Object) 지원**
   * 특정한 프레임워크나 복잡한 규칙에 얽매이지 않는 '순수한 자바 객체'를 뜻함.
   * 복잡한 상속을 받을 필요가 없어서 코드가 엄청 단순해지고 테스트하기도 편해진다.
2. **DI (Dependency Injection, 의존성 주입)**
   * 예전에는 객체를 쓸 때 내가 직접 `new` 키워드로 만들어서 썼다면, 이제는 **스프링 컨테이너가 알아서 필요한 객체를 외부에서 연결(주입)해 주는 방식**. (예: `@Autowired` 사용)
   * 객체들끼리 너무 끈끈하게 결합되는 걸 막아줘서 나중에 코드를 수정하거나 확장하기가 훨씬 좋아진다.
3. **AOP (Aspect Oriented Programming, 관점 지향 프로그래밍)**
   * 로깅, 보안, 트랜잭션같이 여러 곳에서 **공통으로 반복되는 코드를 따로 빼서 모듈화**하는 것.
   * 덕분에 나는 핵심 비즈니스 로직(진짜 중요한 기능) 개발에만 집중할 수 있게 된다!
4. **일관성 있는 트랜잭션(Transaction) 지원**
   * 트랜잭션은 '쪼갤 수 없는 하나의 작업 단위' (전부 성공하거나, 아니면 전부 실패 처리).
   * DB 종류가 바뀌거나 환경이 변해도 스프링이 일관된 방식으로 트랜잭션을 처리해 줘서 코드를 뜯어고칠 필요가 없다.
