# 요청처리 메소드의 파라미터 유형
---

## 학습 목표

- HTTP 요청 조건을 처리하는 어노테이션의 종류, 특징 및 사용 방법 학습
- 요청 경로 처리 (`@PathVariable`)
- 매트릭스 파라미터 처리 (`@MatrixVariable`)
- 요청 파라미터 처리 (`@RequestParam`)

---

## 1. 경로 변수와 `@PathVariable`

### 경로 변수란?

- **경로(Path)**: 서버의 자원을 식별하는 식별자
- **경로 변수(Path Variables)**: 웹 요청 URL에 포함된 파라미터 값을 전달받는 변수
- `@RequestMapping`에 중괄호 `{}`를 사용하여 경로 변수를 명시
- 하나 또는 그 이상의 경로 변수를 포함할 수 있음

```
http://.../cars/red  →  color = "red"
```

### `@PathVariable` 사용법

```java
// 경로 변수 이름을 그대로 사용할 때
@RequestMapping("/cars/{wheel}")
public String method1(@PathVariable String wheel, ...) { ... }

// 경로 변수 이름을 재정의하여 사용할 때
@RequestMapping("/cars/{wheel}")
public String method1(@PathVariable("wheel") String w, ...) { ... }
```

### 단일 경로 변수 예시

```java
@Controller
@RequestMapping("/home")
public class Example01Controller {

    @GetMapping("/exam01/{bookId}")
    public String requestMethod(@PathVariable String bookId, Model model) {
        model.addAttribute("data", "도서 ID: " + bookId);
        return "webpage06";
    }
}
```
- 요청 URL: `http://.../home/exam01/ISBN1234`
- 결과: `도서 ID: ISBN1234`

### 다중 경로 변수 예시

```java
@Controller
@RequestMapping("/home")
public class Example02Controller {

    @GetMapping("/exam02/{category}/publisher/{publisher}")
    public String requestMethod(
            @PathVariable String category,
            @PathVariable String publisher, Model model) {
        model.addAttribute("data", "도서 분야: " + category + "<br/>" + "출판사: " + publisher);
        return "webpage06";
    }
}
```
- 요청 URL: `http://.../home/exam02/IT전문서/publisher/신구문학사`

### 한글 파라미터 처리

HTTP는 영문 대소문자와 일부 기호만 지원하므로, 한글을 전달하려면 **인코딩/디코딩 필터**가 필요하다.

`web.xml`에 아래 필터를 추가한다.

```xml
<filter>
    <filter-name>encodingFilter</filter-name>
    <filter-class>org.springframework.web.filter.CharacterEncodingFilter</filter-class>
    <init-param>
        <param-name>encoding</param-name>
        <param-value>UTF-8</param-value>
    </init-param>
</filter>
<filter-mapping>
    <filter-name>encodingFilter</filter-name>
    <url-pattern>/*</url-pattern>
</filter-mapping>
```

### 실습: 카테고리별 도서 목록 조회

**흐름 요약**

```
URL: /books/{category}
    ↓
BookController.requestBooksByCategory()  ← @PathVariable
    ↓
BookService.getBookListByCategory()
    ↓
BookRepositoryImpl.getBookListByCategory()
    ↓
SQL: SELECT ... FROM book WHERE b_category LIKE '%' || ? || '%'
```

**SQL (H2 DB)**

```sql
SELECT * FROM book WHERE b_category LIKE '%' || ? || '%'
```

**BookRepositoryImpl**

```java
@Override
public List<Book> getBookListByCategory(String category) {
    String sql = "SELECT ... FROM book WHERE b_category LIKE concat('%', ?, '%')";
    return this.template.query(sql, new BookRowMapper(), category);
}
```

**BookController**

```java
@GetMapping("/{category}")
public String requestBooksByCategory(
        @PathVariable("category") String bookCategory, Model model) {
    List<Book> booksByCategory = this.bookService.getBookListByCategory(bookCategory);
    model.addAttribute("bookList", booksByCategory);
    return "books";
}
```

**테스트 URL**
- `http://localhost:8080/books/IT전문서`
- `http://localhost:8080/books/IT활용서`

## 2. 매트릭스 변수와 `@MatrixVariable`

### 매트릭스 변수란?

- 세미콜론(`;`)으로 데이터를 구분하여 URL에 여러 파라미터를 포함하는 방식
- `@RequestMapping` 경로 변수에 `매트릭스_변수=값` 형태로 사용
- 여러 값이 있는 경우 콤마(`,`)로 구분하거나 변수 이름을 반복하여 사용

```
http://.../cars/car;color=red;year=2025  →  {color=red, year=2025}
```

### 주의: Spring 기본 설정에서 매트릭스 변수는 URL에서 제거됨

`servlet-context.xml`에 아래 설정을 추가해야 한다.

```xml
<annotation-driven enable-matrix-variables="true"/>
```

### `@MatrixVariable` 사용법

```java
// 특정 매트릭스 변수를 개별 매개변수로 받을 때
@GetMapping("/cars/{car}")
public String method1(@MatrixVariable("color") String color, ...) { ... }

// 전체 매트릭스 변수를 Map으로 받을 때
@GetMapping("/cars/{car}")
public String method1(@MatrixVariable Map<String, String> color, ...) { ... }
```

### `@MatrixVariable` 속성

| 속성 | 타입 | 기본값 | 설명 |
|---|---|---|---|
| `name` | String | `""` | 매트릭스 변수의 이름 |
| `value` | String | `""` | `name`의 별칭 |
| `pathVar` | String | `DEFAULT_NONE` | 경로 구분이 여러 개일 때 URI 경로 변수의 이름을 명시 |
| `required` | boolean | `true` | 필수 여부 |
| `defaultValue` | String | `DEFAULT_NONE` | 값이 없을 때 사용할 기본값 |

### 예시

```java
@GetMapping("/exam03/{bookId}")
public String requestMethod(
        @PathVariable("bookId") String bookId,
        @MatrixVariable("category") String category, Model model) {
    model.addAttribute("data", "도서 ID: " + bookId + "<br/>도서 분야: " + category);
    return "webpage06";
}
```
- 요청 URL: `http://.../home/exam03/ISBN1234;category=IT전문서`

### 실습: 필터 조건으로 도서 목록 조회

**BookController**

```java
@GetMapping("/filter/{bookFilter}")
public String requestBooksByFilter(
        @MatrixVariable(pathVar = "bookFilter") Map<String, List<String>> bookFilter,
        Model model) {
    Set<Book> booksByFilter = this.bookService.getBookListByFilter(bookFilter);
    model.addAttribute("bookList", booksByFilter);
    return "books";
}
```

**BookRepositoryImpl 핵심 로직**

```java
// publisher 조건으로 검색
String sql = "SELECT ... FROM book WHERE b_publisher LIKE '%' || ? || '%'";

// category 조건으로 검색
String sql = "SELECT ... FROM book WHERE b_category LIKE '%' || ? || '%'";

// 두 결과의 교집합 반환
booksByCategory.retainAll(booksByPublisher);
return booksByCategory;
```

**테스트 URL**
- `http://localhost:8080/books/filter/bookFilter;publisher=길벗;category=IT전문서`
- `http://localhost:8080/books/filter/bookFilter;publisher=길벗;category=IT전문서,IT활용서`

> **⚠️ 주의**: `Set.retainAll()`이 정상 동작하려면 `Book.java`에 `equals()`와 `hashCode()`를 반드시 구현해야 한다. (Eclipse: 마우스 우클릭 → Source → Generate hashCode() and equals()...)

## 3. 요청 파라미터와 `@RequestParam`

### 요청 파라미터란?

- URL의 `?` 뒤에 오는 파라미터
- `파라미터이름=값` 형태, 여러 개일 경우 `&`로 구분

```
http://.../cars?color=red  →  color = "red"
```

### `@RequestParam` 속성

| 속성 | 타입 | 기본값 | 설명 |
|---|---|---|---|
| `name` | String | `""` | 요청 파라미터의 이름 |
| `value` | String | `""` | `name`의 별칭 |
| `required` | boolean | `true` | 필수 여부 |
| `defaultValue` | String | `DEFAULT_NONE` | 값이 없거나 빈 값일 때 사용할 기본값 |

### 실습: 도서 ID로 상세 정보 조회

**BookController**

```java
@GetMapping("/book")
public String requestBookById(
        @RequestParam("id") String bookId, Model model) {
    Book bookById = this.bookService.getBookById(bookId);
    model.addAttribute("book", bookById);
    return "book";
}
```

**BookRepositoryImpl**

```java
@Override
public Book getBookById(String bookId) {
    String sql = "SELECT count(*) FROM book WHERE b_bookId=?";
    int rowCount = this.template.queryForObject(sql, Integer.class, bookId);
    if (rowCount != 0) {
        sql = "SELECT ... FROM book WHERE b_bookId=?";
        return this.template.queryForObject(sql, new BookRowMapper(), bookId);
    }
    throw new IllegalArgumentException("도서 ID가 " + bookId + "인 도서를 찾을 수 없습니다.");
}
```

**books.jsp에 상세정보 링크 추가**

```jsp
<a href="<c:url value="/books/book?id=${book.bookId}"/>" class="btn btn-secondary">
    상세정보 &raquo;
</a>
```

**테스트 흐름**
1. `http://localhost:8080/books` 접속
2. 도서 카드의 "상세정보" 링크 클릭 → 상세 정보 화면 이동
3. "도서 목록" 버튼 클릭 → 목록으로 복귀

## 세 가지 어노테이션 비교 요약

| 어노테이션 | URL 위치 | 구분자 | 사용 예 |
|---|---|---|---|
| `@PathVariable` | 경로 일부 | `/` | `/books/IT전문서` |
| `@MatrixVariable` | 경로 변수 내 | `;` | `/books/filter/bookFilter;publisher=길벗` |
| `@RequestParam` | 쿼리 문자열 | `?`, `&` | `/books/book?id=ISBN1234` |
