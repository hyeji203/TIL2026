# 📘 데이터베이스 연동 복습 정리 (직접 정리)

---

## 🎯 이번 수업에서 배운 것
이번 단원에서는 스프링에서 데이터베이스를 어떻게 연결하는지 전체 흐름을 배웠다.  
단순히 DB를 연결하는 것이 아니라, **Spring JDBC 구조와 역할 분리 방식**이 핵심이다.

---

# 1️⃣ 로깅 (Logging)

## 📌 로깅이란?
프로그램 실행 중 발생하는 정보나 오류를 기록하는 기능이다.  
개발할 때는 디버깅 용도로, 운영 환경에서는 문제 추적에 사용된다.

---

## 📌 Log4j
- Java에서 사용하는 대표적인 로깅 프레임워크
- 성능에 영향을 거의 주지 않으면서 다양한 로그 기능 제공

👉 즉, "프로그램 상태를 기록하는 도구"

---

## 📌 로그 레벨 (중요)

TRACE < DEBUG < INFO < WARN < ERROR < FATAL  

- TRACE: 모든 상세 정보  
- DEBUG: 개발용  
- INFO: 일반 실행 정보  
- WARN: 경고  
- ERROR: 오류  
- FATAL: 치명적 오류  

👉 시험에 자주 나옴

---

## 📌 Slf4j
- 여러 로깅 프레임워크를 하나로 통합해서 사용하는 API
- 스프링에서는 기본적으로 Slf4j 사용

👉 실제 구현은 Log4j, 인터페이스는 Slf4j

---

# 2️⃣ 데이터베이스 (H2)

## 📌 H2 데이터베이스란?
Java 기반의 관계형 데이터베이스(RDBMS)

---

## 📌 특징

- 가볍고 빠름
- JDBC 지원
- 브라우저 콘솔 제공
- 메모리 DB 가능

👉 개발/테스트용으로 많이 사용됨

---

# 3️⃣ Spring JDBC

## 📌 왜 필요한가?
기존 JDBC는 코드가 너무 복잡하다.

예:
- Connection 생성
- Statement 생성
- ResultSet 처리
- 예외 처리
- close()

👉 너무 반복적이고 실수하기 쉬움

---

## 📌 Spring JDBC 역할

✔ DB 연결 관리  
✔ SQL 실행  
✔ 예외 처리  
✔ 자원 자동 해제  

👉 개발자는 "SQL + 결과 처리"만 신경 쓰면 됨

---

# 4️⃣ 전체 구조 이해 (중요)

~~~
Controller → Service → Repository → JdbcTemplate → DB
~~~

👉 실제 흐름  
- Controller: 요청 받음  
- Service: 로직 처리  
- Repository: DB 접근  
- JdbcTemplate: SQL 실행  
- DB: 데이터 저장  

👉 시험 핵심 흐름

---

# 5️⃣ 프로젝트 설정 구조

## 📌 주요 파일 역할

- db.properties → DB 연결 정보  
- root-context.xml → DB 설정  
- servlet-context.xml → JdbcTemplate 설정  
- web.xml → H2 콘솔 설정  
- SQL 파일 → 테이블/데이터 생성  

👉 구조 이해 중요

---

# 6️⃣ DB 연결 설정

## 📌 db.properties

~~~properties
datasource.driverClassName=org.h2.Driver
datasource.url=jdbc:h2:file:C:/webjavaapp/data
datasource.username=admin
datasource.password=manager
~~~

👉 DB 접속 정보 저장

---

## 📌 root-context.xml

### ✔ DataSource 생성

~~~xml
<bean id="dataSource" 
class="org.springframework.jdbc.datasource.DriverManagerDataSource">
    <property name="driverClassName" value="${datasource.driverClassName}" />
    <property name="url" value="${datasource.url}" />
    <property name="username" value="${datasource.username}" />
    <property name="password" value="${datasource.password}" />
</bean>
~~~

👉 DB 연결 객체 생성

---

### ✔ DB 초기화

~~~xml
<jdbc:initialize-database data-source="dataSource">
    <jdbc:script location="classpath:db/create-db.sql" />
    <jdbc:script location="classpath:db/insert-data.sql" />
</jdbc:initialize-database>
~~~

👉 실행 시 자동으로 테이블 + 데이터 생성

---

# 7️⃣ JdbcTemplate

## 📌 핵심 개념
Spring JDBC의 가장 중요한 클래스

👉 역할  
- SQL 실행  
- 결과 처리  

---

## 📌 설정

~~~xml
<bean id="jdbcTemplate"
class="org.springframework.jdbc.core.JdbcTemplate">
    <property name="dataSource" ref="dataSource" />
</bean>
~~~

---

# 8️⃣ 데이터 조회 과정

## 📌 핵심 코드

~~~java
List<Book> list = jdbcTemplate.query(sql, rowMapper);
~~~

👉 동작  
1. SQL 실행  
2. 결과 가져옴  
3. 객체로 변환  

---

# 9️⃣ RowMapper (중요)

## 📌 역할
DB 결과 → Java 객체로 변환

---

## 📌 직접 구현

~~~java
public class BookRowMapper implements RowMapper<Book> {
    @Override
    public Book mapRow(ResultSet rs, int rowNum) throws SQLException {
        Book book = new Book();
        book.setBookId(rs.getString("b_bookId"));
        book.setName(rs.getString("b_name"));
        return book;
    }
}
~~~

👉 한 줄(row) → 객체 하나 생성

---

# 🔟 Repository 구현

~~~java
@Repository
public class BookRepositoryImpl implements BookRepository {

    private JdbcTemplate template;

    @Autowired
    public void setJdbcTemplate(JdbcTemplate template) {
        this.template = template;
    }

    @Override
    public List<Book> getAllBookList() {
        String sql = "select * from book";
        return template.query(sql, new BookRowMapper());
    }
}
~~~

👉 DB 접근 담당 클래스

# 1️⃣1️⃣ 핵심 흐름 정리 (시험 필수)

1. Controller 요청 받음  
2. Service 로직 처리  
3. Repository DB 접근  
4. JdbcTemplate SQL 실행  
5. RowMapper 객체 변환  

---

# 🔥 시험 핵심 포인트

✔ JdbcTemplate = 핵심 클래스  
✔ RowMapper = 결과 매핑  
✔ DataSource = DB 연결  
✔ Spring JDBC = 코드 간소화  
✔ 흐름: Controller → Service → Repository  

---

# 💡 내가 이해한 핵심

👉 기존 JDBC는 "직접 다 처리"  
👉 Spring JDBC는 "자동으로 처리 + 나는 핵심만 작성"
